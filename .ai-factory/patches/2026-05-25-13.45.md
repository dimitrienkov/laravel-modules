# Breaking-clean review fix: cache diagnostics, typed VOs, API renames, manifest split

**Date:** 2026-05-25 13:45
**Files:** 30+ files across src/, tests/, fixtures
**Severity:** high

## Problem

Review of `src/` uncovered runtime bugs in cache commands (ignored write/delete results), non-deterministic loader pipeline sort, weak config validation in scanner, silent duplicate enum option handling, inconsistent public API naming, legacy list-form dependency sugar, and responsibility bloat in ManifestValidator/ManifestRepository.

## Root Cause

Multiple independent issues accumulated during rapid v2.0 development:
- Cache write/delete results not checked — success reported even on I/O failure.
- `usort()` without stable tie-breaker gave unpredictable loader order for equal priorities.
- `ModuleDirectoryScanner` silently skipped invalid config entries and allowed traversal outside `app_path()`.
- `ModuleDependencies` supported two input forms (list + map) — unnecessary complexity for a breaking v2.0.
- `FeatureDefinitionFactory` silently deduplicated options via `array_unique()`.
- Public API used abbreviated names (`bool()`, `Int`, `Bool`) that read poorly.
- `ModuleManifestRepository` had a redundant `updateFeatureValues()` alias and mixed JSON reading with business logic.
- `ManifestValidator` handled both top-level and settings validation in one class.

## Solution

Breaking-clean fixes applied without backward compatibility:

1. **Cache diagnostics**: New `ModuleCacheWriteException`/`InvalidModuleCacheException`. `ModuleRegistryCache::write()` checks byte count, `forget()` checks unlink result. Commands return FAILURE on error.
2. **Cache payload VO**: `ModuleRegistryCachePayload` + `CachedModuleDescriptor` with strict invariant validation (no duplicates in load_order, exact match between modules and load_order).
3. **Stable pipeline sort**: tie-break by registration index preserves insertion order for equal priorities.
4. **Scanner validation**: `InvalidConfigurationException` for non-string/empty entries. `realpath()` boundary check against `app_path()`.
5. **Dependencies**: Removed list-form support. `ModuleDependency` VO wraps each entry.
6. **Duplicate options**: Throws `InvalidManifestException` instead of silent `array_unique()`.
7. **API renames**: `bool()` → `getBool()`, `int()` → `getInt()`, `string()` → `getString()`.
8. **Enum cases**: `Bool` → `Boolean`, `Int` → `Integer`.
9. **Manifest repository**: Removed `updateFeatureValues()` and generic `save()`. Single `saveValues()` method. Extracted `ManifestDocumentReader` for JSON I/O.
10. **Validator split**: `ManifestSettingsValidator` handles `settings.schema`/`settings.values` subtree.
11. **Service provider**: Split `registerCoreBindings()` into 4 focused methods.

## Prevention

- Always check return values of filesystem operations (write, delete, rename).
- Use stable sort or explicit tie-breaker when determinism matters.
- Validate configuration inputs at system boundary, don't silently skip.
- In a breaking version, remove legacy sugar rather than maintaining two code paths.
- Silent normalization (dedup, coercion) hides manifest errors — prefer explicit validation.
- Split validators when they handle structurally independent subtrees.
