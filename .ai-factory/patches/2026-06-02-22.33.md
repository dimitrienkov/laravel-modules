# Module-aware generators: listener event repoint, migration --path guard, ModuleSegment single owner

**Date:** 2026-06-02 22:33
**Files:** src/Console/Commands/Make/{MakeListener,MakeMigration,MakeModel,MakeObserver,MakePolicy,MakeEvent,MakeMiddleware,MakeRequest,MakeConsoleCommand,MakeFactory,MakeSeeder,MakeComponent,MakeController,MakeMail}.php, src/Console/Concerns/{ModuleAwareGenerator,ArchitecturalGenerator}.php, src/Console/Support/ModuleOption.php (new), src/Support/{ModuleSegment.php (new),ModuleLayout.php}, src/Application/Enums/ScaffoldComponent.php, src/Application/Support/ModuleSkeletonBuilder.php, src/Providers/ModuleLoaderServiceProvider.php, AGENTS.md, plus tests
**Severity:** medium

## Problem

A branch review of `feature/make-module-generators` surfaced one correctness
asymmetry plus a set of robustness/observability and quality findings:

- `make:listener PaymentListener --event=PaymentReceived --module=blog` generated
  a listener importing the **host** `App\Events\PaymentReceived`, while every
  other paired generator (controller/model/factory/component/mail) already
  repoints host references into the module. The listener silently referenced an
  event the module loader never registered.
- `make:migration --module=X --path=foo` silently overwrote the user's `--path`
  with the module migrations directory (silent input mutation).
- The same structural sub-path (`Domain/Models`, `Database/Factories`, …) was
  spelled out as a literal in 3–5 places: `ModuleLayout` getters, each
  generator's `moduleSubNamespace()`, `ScaffoldComponent`, and
  `ModuleSkeletonBuilder` — a multi-source-of-truth drift risk.
- Test gaps: no negative side on the view-repoint guards, no case-insensitivity
  coverage for path-overriding generators, partial matching-test matrix, no
  `makeDirectory` idempotency / architectural-no-leak / empty-interactive cases.
- `AGENTS.md` still marked the (now implemented) `make:* --module` generators as
  roadmap.

## Root Cause

- **Listener asymmetry:** Laravel's `ListenerMakeCommand::buildClass` hard-codes
  `App\Events\…` via the `{{ eventNamespace }}` token, auto-qualifying any bare
  `--event`. The module-aware subclass overrode namespace/path hooks but not
  `buildClass`, so the event import never followed the class into the module.
- **`--path` mutation:** `MakeMigration::handle()` unconditionally called
  `setOption('path', …)` in module mode without checking whether the user had
  already supplied one — check-then-act with no conflict guard.
- **Segment triplication:** there was no single owner for the runtime/generator
  structural segments. `ModuleLayer` deliberately owns only the *architectural*
  layers, so the structural ones had no home and were copied per call-site.

## Solution

- **A1:** added `ModuleLayout::eventsNamespace()` and overrode
  `MakeListener::buildClass` to post-process `parent::buildClass()` — replacing
  the host `App\Events\` prefix with the module events namespace **only** when
  the event was auto-qualified (mirroring the parent's own
  `Str::startsWith([appNamespace, 'Illuminate', '\\'])` test). A fully-qualified
  `--event` is left untouched. No native method body was copied.
- **B2:** `MakeMigration::handle()` now fails fast with a clear message when
  `--module` and an explicit `--path` are combined, before any file write.
- **B1 (documentation only):** the `parent::handle(); return SUCCESS` exit
  contract was left as-is and its comment tightened. The originally-planned
  forward-compat `is_int($result)` passthrough is **infeasible** under PHPStan
  level 8 (no baseline): `GeneratorCommand::handle()` is `@return bool|null` and
  `MigrateMakeCommand::handle()` is `void`, so an int branch is provably dead and
  PHPStan rejects it. Behaviour and host parity are unchanged, which is what B1's
  acceptance actually requires.
- **C2 (keep + pin):** delegating `MakeFactory::buildClass` to the parent was
  evaluated and rejected — the parent derives `{{ factoryNamespace }}` from
  `rootNamespace` via `replaceFirst`, which double-mangles once the trait's
  `getDefaultNamespace` returns the module namespace. The reimplementation is
  kept and a full token-pinning regression test was added.
- **D1:** introduced `Support/ModuleSegment` (sibling of `ModuleLayer`) as the
  single owner of structural segments duplicated across ≥2 call-sites, and routed
  `ModuleLayout`, the relevant `moduleSubNamespace()` returns, `ScaffoldComponent`
  and `ModuleSkeletonBuilder` through it. Single-token paths (`Config`, `Routes`,
  `Providers`) and one-offs stayed literals — a one-token directory has no
  separator-form ambiguity, so centralising it would add indirection without
  removing real duplication.
- **D2/D3:** `ModuleLoaderServiceProvider::packageStubsPath()`,
  `ModuleLayout::classFilePath()`, a `MATCHING_TEST_OPTIONS` trait constant,
  `Console\Support\ModuleOption::make()` for the shared `--module` option, plus
  small readability fixes (guard clauses, `moduleResolved`→`isModuleResolved`,
  docblock FQCN).

## Prevention

- **Paired generators are a class of bug:** whenever a generator repoints any
  host reference into a module (namespace, view, request, factory), check every
  *other* host reference the same stub can carry — events, listeners, models. A
  symmetry checklist across `Console/Commands/Make/*` catches the next one.
- **Mirror the parent's own condition, don't reinvent it:** the listener repoint
  reuses Laravel's exact `startsWith` auto-qualification test, so FQCN events
  behave identically to native. Post-process `parent::…()` instead of copying
  bodies.
- **Never silently overwrite user-supplied options** in a command adapter; a
  conflicting flag is a fail-fast error.
- **Forward-compat code must be reachable under the typed contract.** Before
  adding a "future Laravel might return X" branch, check the parent's declared
  return type — PHPStan level-8 (no baseline) treats `@return` as certain and
  rejects provably-dead `is_*` narrowing.
- **One owner per repeated structural string.** New module sub-paths that recur
  in more than one call-site belong in `ModuleSegment` (structural) or
  `ModuleLayer` (architectural), never re-spelled inline.
- **Guard tests pin both sides of a text substitution** (module string present
  AND host string absent) so a future stub-literal change fails loudly instead of
  becoming a silent no-op.
