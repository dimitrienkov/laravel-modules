# Lifecycle review fixes: ConfirmableTrait, RemoveStrategy enum, infrastructure hardening

**Date:** 2026-05-27 02:45
**Files:** 18 src/ files, 16 test files, 3 new test files
**Severity:** medium

## Problem

Code review of the `feature/lifecycle-usecases-commands` branch found 2 critical + 19 warning issues across commands, application layer, infrastructure, and tests: non-idiomatic production guards, boolean flag in public API, TOCTOU race in state deletion, missing `$previous` in exception chains, magic numbers, data-carrier with cleanup responsibility, and missing test coverage.

## Root Cause

Multiple independent design shortcomings accumulated during initial lifecycle feature implementation:

- **Boolean flag in public API:** `RemoveModuleUseCase::execute(bool $deletePermanently)` violated `code-quality.md` §18 — boolean flags that switch method behavior should be enums.
- **Non-idiomatic production guard:** `ModulesRemoveCommand` used manual `confirm()` instead of Laravel's `ConfirmableTrait` + `--force` pattern. `ModulesInstallCommand` had no production guard at all.
- **`--force` collision:** `MakeModuleCommand` used `--force` for "overwrite existing module" which conflicts with `ConfirmableTrait`'s reserved `--force` semantics.
- **TOCTOU in state deletion:** `isFile()` + `delete()` sequence in `ModuleStateRepository::delete()` was vulnerable to race conditions; `deleteFileIfExists()` already existed but wasn't used.
- **Missing `$previous` in exception chain:** `InvalidManifestException::forPath()` and `ManifestDocumentReader` lost `JsonException` context.
- **Data-carrier with behavior:** `PreparedSource` held `LocalFilesystem` dependency and `cleanup()` method — cleanup responsibility belonged to `ModuleSourcePreparer`.
- **Magic number:** `dirname(__DIR__, 3)` in `ModuleSkeletonBuilder` was fragile if class moved.

## Solution

- Created `RemoveStrategy` enum replacing boolean parameter.
- Applied `ConfirmableTrait` to `ModulesRemoveCommand` and `ModulesInstallCommand`.
- Renamed `MakeModuleCommand --force` to `--overwrite`.
- Added `--enabled/--disabled` conflict detection in `ModulesListCommand`.
- Used `deleteFileIfExists()` to eliminate TOCTOU.
- Added `?Throwable $previous` to `InvalidManifestException::forPath()`.
- Moved cleanup from `PreparedSource` to `ModuleSourcePreparer::cleanup()`.
- Injected `stubsPath` via constructor in `ModuleSkeletonBuilder`.
- Added explicit factory for `AtomicJsonWriter` in service provider.
- Added try/catch around `deleteDirectory` in `restoreBackup`.
- Added `@unlink` in `AtomicFileWriter` finally block.
- Added 3 architecture tests (lifecycle commands, exceptions, providers dependency).
- Added 25+ new test cases covering ListModulesUseCase, PartialModuleRollback, ModuleSkeletonBuilder, ModuleState factory methods, ModuleStateRepository write operations, install with dependencies.
- Deduplicated `deleteDirectory()` helper across 12 test files.

## Prevention

- Use `ConfirmableTrait` for all destructive Artisan commands from the start.
- Reserve `--force` for `ConfirmableTrait`; use `--overwrite`, `--yes`, etc. for other boolean options.
- Replace boolean behavior flags with enums in public API signatures.
- Always pass `$previous` when wrapping exceptions in factory methods.
- Keep data-carriers (DTOs/VOs) free of service dependencies and behavior.
- Inject paths explicitly rather than using relative `dirname()` magic numbers.
- Add architecture tests alongside new commands to enforce dependency boundaries.
