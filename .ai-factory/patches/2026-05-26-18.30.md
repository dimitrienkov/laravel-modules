# Post-review lifecycle branch fixes: contracts, DI, naming, security

**Date:** 2026-05-26 18:30
**Files:** ~40 files across src/ and tests/
**Severity:** high

## Problem

Code review of `feature/lifecycle-usecases-commands` branch identified 4 blockers, ~15 medium-severity issues and ~15 low-severity improvements across contracts, architecture, naming, security and test coverage.

Key issues: missing rollback in RemoveModuleUseCase, unchecked return values, no error handling in ModulesListCommand, mixed raw PHP and Laravel Filesystem, lifecycle-specific exceptions in infrastructure layer, duplicate utility methods, negative boolean parameters, incomplete zip traversal check, missing ConfirmableTrait on destructive commands.

## Root Cause

Fast-paced feature development without sufficient contract design review. Infrastructure layer (ModuleDirectoryOperations, ZipExtractor, ModuleStateRepository) accumulated raw PHP I/O calls instead of using injected Filesystem. Naming evolved organically without consistency pass.

## Solution

30-step refactoring across 7 phases:

1. **Blockers:** Removed TemporaryDirectoryCleaner (unified via Filesystem DI), fixed cleanupDirectory return value, added rollback in RemoveModuleUseCase, added try-catch in ModulesListCommand
2. **Contracts:** Added has()/reset() to ModuleRegistryInterface, created ModuleRegistryCacheInterface, decoupled DirectoryOperationException from lifecycle exceptions, renamed write methods (writeDocument/writeState/writeValues)
3. **Filesystem migration:** Injected Filesystem into ModuleStateRepository, refactored extractValues with assertJsonObject helper
4. **Value objects:** Added ModuleState named constructors (initialState/updatedFrom/withEnabled), extracted PathNormalizer, replaced studlyCase with Str::studly, created ModuleFileNames/ModulePermissions constants, renamed $disabled to $enabled, removed toManifestArray duplicate
5. **Security:** Added null-byte check in zip traversal, ConfirmableTrait on destructive commands, throw on missing provider stub
6. **Tests:** Fixed filter assertions, added disableFailsWhenModuleNotFound, fixed semver constraint, narrowed architecture allowedClasses
7. **Naming:** Renamed ModuleLifecyclePaths to ModuleDirectoryPaths

## Prevention

- Contract design review before implementation: define interfaces first, then implement
- Infrastructure classes should always use injected abstractions (Filesystem), never raw PHP I/O
- Named constructors for VOs that require timestamp consistency
- Negative booleans ($disabled) should be caught in PR review
