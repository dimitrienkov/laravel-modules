# Diagnostic logger: make it unbreakable and unsilenceable, then de-duplicate the event taxonomy

**Date:** 2026-06-02 13:30
**Files:** src/Support/Logging/ModuleLogger.php, src/Support/Logging/LogEvent.php (new), src/Support/Logging/LifecyclePhase.php (new), src/Support/ClassName.php (new), src/Contracts/ModuleDiagnosticsInterface.php, src/Support/Logging/NullModuleDiagnostics.php, src/Loaders/VO/LoadReport.php, src/Loaders/VO/PipelineRunSummary.php, src/Loaders/Pipeline/ModuleLoaderPipeline.php, src/Loaders/RouteLoader.php, src/Loaders/FactoryLoader.php, src/Loaders/ServiceProviderLoader.php, docs/logging.md, README.MD, plus 12 test files
**Severity:** medium

## Problem

A branch review (self + dandy-* lenses) of the new opt-in diagnostic logging surfaced two
substantive correctness gaps and a set of quality/test-coverage gaps — no hard blockers,
phpstan/format/test were green:

1. **Resilience:** `ModuleLogger::*` runs on `ModuleLoaderServiceProvider::boot()`. The actual
   `$this->logger->log(...)` write was not isolated, so a throwing host log channel (disk full,
   permission denied) would propagate out of `boot()` and break the app — violating the "diagnostics
   are additive, never break boot" intent.
2. **Failure visibility:** `shouldLog()` let ERROR-level events bypass the per-category toggle but
   still applied the global threshold, so `MODULES_LOG_LEVEL=emergency` silently swallowed
   `pipeline.loader.failed` / `lifecycle.{op}.failed`. The class docstring promised "a failure
   cannot be hidden" — an internal contradiction.
3. **Coverage/quality:** lifecycle happy-path terminal invariants, `rolled_back`/`backup_created`
   and `cacheHit`/`cacheMiss` events were unasserted; event-name string literals were duplicated
   between the producer and two test files with no compiler protection.

## Root Cause

- **Resilience:** the write was inlined at all 18 emit sites; there was no single seam to wrap, so
  isolation was never applied. Observability code that can throw must have exactly one guarded exit.
- **Visibility:** the threshold floor was conceptually "errors bypass the *category*" but the code
  reused the same `priority >= threshold` comparison for the error branch, conflating "category
  bypass" with "threshold bypass". The fix is a true floor: `priority >= ERROR ⇒ always record`.
- **Literals:** the taxonomy (`category.subject.verb`, `lifecycle.{op}.{phase}`) lived only as
  inline strings — a typo could drift producer and parser apart undetected.

## Solution

- **T1:** introduced a private `write(level, message, context)` that wraps the single PSR-3 call in
  `try { ... } catch (\Throwable) { /* deliberately silent — host ExceptionHandler stays the source
  of truth */ }`. All 18 emit sites route through it. `report()` ordering in the pipeline is
  untouched, so a broken channel never masks the domain exception.
- **T2/T9:** `shouldLog()` now returns `true` for any event at `priority >= ERROR` (floor), bypassing
  both the category toggle and the threshold while logging is enabled. `discovery.root.rejected`
  (an aborting condition that precedes a thrown exception) was promoted from WARNING to ERROR so it
  rides the same floor. Docstring + docs/logging.md updated to match.
- **T4:** normalised the threshold to a priority once in the constructor (out of the hot path);
  extracted an `isCategoryEnabled()` positive predicate; `priority($this->level)` is now computed
  zero times per call.
- **T5/T7:** extracted `LogEvent` (string-backed enum, 14 fixed names + `lifecycle(op, phase)` helper)
  + `LifecyclePhase` enum, and a pure `ClassName::short()` helper (replaces three duplicated
  class-basename one-liners). Producer and tests now reference symbols; a feature test still asserts
  the raw strings, proving the enum values are byte-identical.
- **T6:** renamed `loaderOutcome` → `loaderCompleted` (symmetry with `loaderFailed`) and added
  `LoadReport::wasSkipped()` (positive predicate paired with `wasApplied()`).
- **T3:** added happy-path terminal-invariant spy tests for all 8 lifecycle UseCases
  (started ×1 / succeeded ×1 / failed ×0), `rolled_back`/`backup_created` asserts on Update+Remove
  compensating paths, `cacheHit`/`cacheMiss` on warm/cold registry loads, and a gating-matrix cell.

## Prevention

- **Isolate any I/O that observability performs behind one guarded method.** If logging/metrics can
  throw, the call site must not be able to break the host. One seam, one `catch`, a "why" comment.
- **A failure-visibility contract needs a true floor, not a bypass.** If "errors can't be hidden",
  errors must ignore *every* suppressor (category AND threshold), and a test must assert it at the
  highest threshold with the category off — otherwise the contract is aspirational.
- **Model a repeated cross-file string taxonomy as a string-backed enum**, and keep one test that
  pins the enum value to the exact wire string so refactors can't silently change the contract.
- **When adding non-impl types (enums/VO) to a namespace guarded by an `arch()->classes()->toImplement()`
  invariant, expect the selector to include enums** — refine the arch test with `->ignoring([...])`
  rather than relocating the type away from where it belongs.
- For lifecycle use cases, assert the *happy-path* terminal invariant (exactly-once started/succeeded,
  never failed) with `Mockery::spy`, not just the failure branch — a missing success emit is invisible
  otherwise.
