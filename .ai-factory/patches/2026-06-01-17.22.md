# Consolidated manifest/state contract invariants and aligned list/group behavior

**Date:** 2026-06-01 17:22
**Files:** src/Support/StringKeyedObject.php, src/Manifest/Parsing/ManifestFieldReader.php, src/Registry/VO/CachedModuleDescriptor.php, src/Manifest/ModuleStateRepository.php, src/Manifest/VO/ModuleOrigin.php, src/Manifest/VO/ModuleStateDocument.php, src/Manifest/VO/Checksum.php, src/Application/Support/ModuleSourcePreparer.php, src/Console/Commands/Modules/ModulesListCommand.php, src/Application/Support/ModuleGroupLabelResolver.php, src/Application/UseCases/ScaffoldModuleUseCase.php, src/Application/Enums/ModuleSourceKind.php, plus tests and docs
**Severity:** medium

## Problem

Accumulated contract drift and duplicated invariants across the manifest/state layer:
the "string-keyed JSON object" loop was copy-pasted in three places; the kind↔checksum
rule lived in three spots (`ModuleOriginKind::requiresChecksum()`, the `ModuleOrigin`
constructor, and `parseChecksum()`); `modules:list --group` accepted any string while
`--kind` fail-fast validated; the group regex allowed `foo-`/`foo--bar`; `ModuleStateDocument::$origin`
serialized to a `source` key (name/serialization mismatch); `writeValues()` wrote
`defaultDisabled()` when no state file existed (silently flipping `enabled`); group label
config failed silently on malformed input; and docs implied the provenance checksum was
integrity verification.

## Root Cause

These were not runtime crashes but latent contract weaknesses: duplicated logic drifts,
multiple owners of one invariant produce inconsistent error messages, lenient input
validation hides typos, and a name/serialization mismatch (`origin` field → `source` key)
plus a silent state fallback are exactly the kind of "works until it doesn't" issues that
surface only under specific install/scaffold ordering.

## Solution

- Extracted one `StringKeyedObject::toStringKeyedObject(array, callable(): Throwable)` helper;
  callers keep their boundary "is this a JSON object" check and pass a closure that builds
  their own typed exception, so the shared helper stays free of any exception class and each
  call site preserves its exact message. Removed the now-dead private `requireStringKeys()`.
- Made the constructor the **single owner** of the kind↔checksum invariant; `parseChecksum()`
  parses only, and `fromArray()` wraps the constructor's `InvalidArgumentException` into a
  state-path-scoped `InvalidModuleStateException`.
- Tightened the group regex to segment kebab-case `/^[a-z0-9]+(-[a-z0-9]+)*$/` and made
  `modules:list --group` fail-fast on bad format with a group-specific empty message.
- Renamed `ModuleStateDocument::$origin` → `$source` (no alias/migration; JSON key unchanged).
- `writeValues()` now persists the passed module's own state when no state file exists.
- `ModuleGroupLabelResolver` is fail-loud on malformed config but treats an **absent** map
  (null) and a missing code as allowed bare-code fallbacks.
- Documented the checksum as read-time provenance, not integrity verification (roadmap).

## Prevention

- **One invariant, one owner.** When the same domain rule (e.g. presence/absence tied to an
  enum) appears in a constructor and a parser/factory, keep it in the constructor and let
  factories translate/contextualize the thrown error — don't re-check it.
- **Deduplicate boundary loops via a callable-injecting helper**, not a base class or a
  helper that throws a fixed exception type — that keeps each call site's typed exception and
  message intact while removing the duplication.
- **Watch for absent vs malformed config.** A strict "must be an array" check must still let
  `null`/unset through as the default-empty case, or it breaks every host that never set the
  key. Distinguish "absent (allowed default)" from "present but wrong type (fail-loud)".
- **Field name should match its serialized key** (`$source` ↔ `"source"`); a mismatch is a
  silent readability/debugging tax.
- **A "write subset" method must not reset the rest of the document** — when no file exists,
  preserve the caller's state instead of a hard default.
- After segment-pattern changes, grep **all** test data providers (not just the obvious test
  file) for old-contract cases — `ManifestMetaTest` encoded the leading-digit rejection that
  the new regex intentionally relaxed.
