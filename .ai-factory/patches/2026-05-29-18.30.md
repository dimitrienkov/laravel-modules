**Severity:** medium

## Problem

Review of `feature/meta-group-source-origin` surfaced one real defect plus several
hardening gaps:

- **F1 (real bug):** `make:module x --group="Bad Group"` produced
  `Failed to scaffold module [Bad Group]: ...` — the invalid *group* value was
  rendered in the *module name* slot of `ModuleScaffoldException::forModule()`.
- **F2:** the `zip ⇒ checksum` / `local ⇒ no checksum` provenance invariant lived
  only in factory convention; `checksum` was a bare `?string` (no hex validation,
  magic `'sha256'` literal in `ModuleSourcePreparer`, raw `hash_file` outside infra).
- **F3:** `config('modules.groups')` had no consumer — `modules:list` printed the
  raw group code, never the configured label.
- **F4–F6:** two near-identical enums with no docblock distinguishing their axes,
  nested guards / param name drift in `ModuleOrigin::fromArray`, and a silent
  `catch (Throwable) {}` in cleanup.
- Test gaps: `source` as list/scalar, empty/whitespace `installed_version`, empty
  `kind`; a non-deterministic cleanup test asserting `filemtime($d) > time() - 5`.

## Root Cause

- **Continuation bias** in `validateGroup`: the method was copied from `validateName`
  but the group value was passed where the factory expected `$moduleName`. The
  factory signature `forModule(string $moduleName, ...)` does not stop a wrong-but-
  same-typed argument, so the mistake compiled and "looked right".
- **Invariant by convention, not by type:** because `checksum` was `?string` and the
  pairing rule was only enforced inside the named factories, every other path
  (`__construct`, `fromArray`) could silently produce an illegal `ModuleOrigin`.
  Primitive obsession (`?string`) hid both the format contract (64-hex sha256) and
  the kind↔checksum coupling.
- **Half-built feature:** the `modules.groups` config slot was added without the
  read-side resolver, so the contract existed but did nothing.

## Solution

- **F1:** `validateGroup(string $name, ?string $group)` now passes the module name to
  `forModule()` and embeds the group in the reason text
  (`invalid module group [{$group}] — must be kebab-case.`), forwarding `$e` as
  `previous`. Aligned the inner `field` to `meta.name` / `meta.group`.
- **F2:** introduced a single-value `final readonly Checksum` VO (64-hex lowercase
  sha256, validated in the constructor; algorithm is the concept constant
  `Checksum::ALGORITHM`, never persisted, no `equals`/`fromHex`/`toHex` — no
  consumer). Hashing moved into `LocalFilesystem::hashFile(): string|false` (the
  `false` guard is preserved so an unreadable archive still raises
  `ModuleSourceException`, not a `TypeError`). The kind↔checksum invariant is now
  **kind-driven** via `ModuleOriginKind::requiresChecksum()` (exhaustive `match`),
  enforced in both the constructor (`InvalidArgumentException` — programmer error)
  and `fromArray` (`InvalidModuleStateException` — untrusted state.json), validated
  *before* construction. `state.json` format is unchanged (bare hex).
- **F3:** added `ModuleGroupLabelResolver` (injected `Config\Repository`, lenient at
  display time) and wired it into `ModulesListCommand` to render `"Label (code)"`.
- **F4–F6:** docblocks separate the two axes (`ModuleSourceKind` = staging-input
  format vs `ModuleOriginKind` = fixed provenance); `fromArray` param renamed to
  `$statePath` with flat guards + a `parseChecksum` helper; cleanup's empty catch
  now carries an explicit best-effort comment (no logger introduced — project
  forbids runtime logging in `src/`).

## Prevention

- When copy-adapting a sibling validator/method, verify every *positional* argument
  to factories — same-typed slots (here two `string`s) make misplacement invisible
  to the type checker. Prefer named arguments at such call sites.
- Encode cross-field invariants in the type system, not in factory convention: a
  VO must reject illegal states in its constructor so `fromArray`, `new`, and
  factories cannot diverge. Express enum-coupled rules through an exhaustive `match`
  method on the enum so a future case fails the build instead of slipping through.
- Replace primitive `?string` domain values (digests, ids) with validated VOs to
  pull format rules and magic literals (`'sha256'`) to one owner.
- When adding a config contract, add its consumer in the same change or mark it
  roadmap — an unread config key reads as dead code.
- Never assert on wall-clock / `time()` in cleanup tests; diff the before/after set
  of artifacts instead (deterministic, order-independent).
