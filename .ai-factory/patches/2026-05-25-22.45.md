# Batch fix: review findings from phase-2-new-loaders

**Date:** 2026-05-25 22:45
**Files:** src/Loaders/FactoryLoader.php, src/Loaders/BroadcastLoader.php, src/Support/AtomicFileWriter.php, src/Support/ModuleLayout.php, src/Loaders/ObserverLoader.php, src/Loaders/PolicyLoader.php, src/Loaders/MiddlewareLoader.php, src/Loaders/BladeComponentLoader.php, src/Registry/ModuleRegistryCache.php
**Severity:** medium

## Problem

Code review of phase-2 loaders identified 5 actionable issues:
1. FactoryLoader uses bare ReflectionProperty on Factory::$factoryNameResolver — breaks if Laravel changes the property name/visibility.
2. BroadcastLoader uses `$app->booted(require ...)` — fails fatally if host app has no BroadcastManager.
3. AtomicFileWriter creates `.lock` files but never deletes them — accumulates orphaned locks.
4. Five loaders hardcode namespace segments (`Domain\\Observers`, `Http\\Middleware`, etc.) instead of resolving through ModuleLayout.
5. ModuleRegistryCache::forget() has TOCTOU race between is_file() and unlink().

## Root Cause

1. No defensive wrapper around Reflection API usage — assumed internal Laravel property is stable.
2. `$app->booted()` always fires regardless of whether broadcasting is configured — bare `require` runs unconditionally.
3. Lock cleanup was simply missing from the finally block — the file handle was closed but the .lock inode was left behind.
4. Namespace segments were inlined during initial loader implementation rather than centralized in ModuleLayout, which only had path methods.
5. Standard TOCTOU pattern: another process can delete the file between the existence check and the unlink call.

## Solution

1. Wrapped `currentFactoryNameResolver()` in try/catch for ReflectionException, returning null on failure (safe — fallback to `defaultFactoryClassFor()`).
2. Replaced `$app->booted()` + `require` with `ContainerLifecycleHooks::callAfterResolving(BroadcastManager::class, ...)` — channels.php only loads when BroadcastManager is actually resolved.
3. Added `@unlink($path . '.lock')` in the finally block after fclose.
4. Added 6 namespace methods to ModuleLayout (bladeComponentNamespace, factoryNamespace, middlewareNamespace, modelNamespace, observerNamespace, policyNamespace) and updated all 5 loaders to use them.
5. Changed `forget()` to `@unlink($path)` followed by `is_file()` check — succeeds whether we deleted or another process did.

## Prevention

- When using Reflection on framework internals, always wrap in try/catch — internal APIs are not stable contracts.
- For deferred loading, prefer `callAfterResolving` over `booted()` — it naturally guards against the service being absent.
- Any filesystem operation that creates temp/lock files should clean up in the same finally block.
- Namespace segments should be centralized in ModuleLayout alongside path methods — loaders.md rule "resolve paths through ModuleLayout" extends to namespaces.
- For atomic file operations, prefer `@unlink` + existence re-check over check-then-unlink to avoid TOCTOU races.
