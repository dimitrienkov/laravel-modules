# Structure drift: tests and commands not mirrored after src refactor

**Date:** 2026-05-25 01:25
**Files:** tests/Unit/Manifest/VO/*, tests/Unit/Manifest/Parsing/*, tests/Unit/MoonShine/MoonShineModuleAutoloaderTest.php, src/Console/Commands/Modules/*, src/Providers/ModuleLoaderServiceProvider.php, tests/Feature/ModulesOptimizeCommandTest.php, tests/Architecture/CommandsStructureTest.php, AGENTS.md, .ai-factory/ARCHITECTURE.md
**Severity:** low

## Problem

After refactoring `src/Manifest/` into subfolders (VO/, Parsing/, Enums/), the corresponding test files remained flat in `tests/Unit/Manifest/`. Similarly, `MoonShineLoaderTest` stayed in `tests/Unit/Loaders/` despite the class moving to `src/MoonShine/`, and lifecycle commands stayed in root `src/Console/Commands/` instead of the planned `Modules/` subdirectory.

## Root Cause

The original refactoring PR moved source classes into subdirectories but did not propagate the structural change to corresponding tests. This is a common pattern when refactoring is done incrementally — the "mirror structure" convention between `src/` and `tests/` is implicit and easy to forget when the rename touches only production code.

The commands case is slightly different: `ARCHITECTURE.md` already prescribed `Commands/Modules/` but the files were never physically moved during the initial commit.

## Solution

1. `git mv` / `mv` all affected test files into matching subdirectories
2. Updated `namespace` declarations in every moved file (PSR-4 compliance)
3. Moved commands to `src/Console/Commands/Modules/` and updated imports in the service provider and feature tests
4. Renamed `MoonShineLoaderTest` class to `MoonShineModuleAutoloaderTest` to match the actual SUT
5. Added `CommandsStructureTest` architecture guardrail to prevent future drift
6. Updated `AGENTS.md` and `ARCHITECTURE.md` to reflect the actual tree

## Prevention

- When moving source classes into subdirectories, always check `tests/` mirror in the same PR
- Add architecture tests that enforce structural conventions (like the new `CommandsStructureTest`)
- After any namespace refactoring, run `grep -r "old\\namespace"` across the whole repo before committing
- Keep `AGENTS.md` and `ARCHITECTURE.md` in sync with actual file tree as part of the refactoring checklist
