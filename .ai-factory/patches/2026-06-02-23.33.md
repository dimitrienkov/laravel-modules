# Static-analysis hardening: parallelism, strict-rules, type-coverage, PER-CS 3.0

**Date:** 2026-06-02 23:33
**Files:** phpstan.neon.dist, rector.php, .php-cs-fixer.dist.php, composer.json, 24 src/ files (+rector/format sweep over 98 files)
**Severity:** medium

## Problem

Two distinct issues plus a quality push:

1. **Parallelism silently disabled.** `composer phpstan`/`rector`/`format` all ran single-threaded on a
   16-core box. `phpstan analyse --debug` and `rector process --debug` force single-process; php-cs-fixer
   `fix --sequential` disables its parallel runner. `phpstan.neon` even declared `parallel.maximumNumberOfProcesses: 4`
   which never took effect because `--debug` overrode it. The flags were added during a big refactor
   (commit 88d3c17) to surface a crashing parallel worker's real stack trace, then left in as cargo-cult.
2. **Coverage below "TS-level".** No `phpstan-strict-rules`, no type-coverage measurement, no `bleedingEdge`,
   PSR-12 instead of PER-CS 3.0. Popular packages (MoonShine level 5, nwidart, spatie/laravel-data level 5)
   are *less* strict, so they were no benchmark.

## Root Cause

- `--debug`/`--sequential` are debugging aids, not production defaults — they trade ~10x throughput for a
  readable single-process trace. The original parallel-worker crash was already fixed in code; the flags
  outlived their reason.
- Strictness gaps were simply unconfigured tooling, not code debt: enabling everything surfaced only
  **93 distinct locations** (the raw "247 errors" were type-coverage emitting the same aggregate at every
  missing-return site — 151 of them collapsed to 6 methods).

## Solution

- **composer.json**: dropped `--debug`/`--sequential`/`--clear-cache` from default scripts; added
  `phpstan:debug`, `rector:debug`, `rector:fresh` escape hatches. Parallelism now on by default.
- **phpstan.neon.dist**: `bleedingEdge`, `phpstan-strict-rules` (allRules), `phpstan-deprecation-rules`,
  `tomasvotruba/type-coverage` at 100% (return/param/property/constant/declare), `checkUninitializedProperties`,
  `checkBenevolentUnionTypes`, `parallel.max 4→16`. Registered via `phpstan/extension-installer` (so the manual
  larastan include was removed to avoid a double-include).
- **rector.php**: `STRICT_BOOLEANS`, `INSTANCEOF`, `withPhpSets(php83)`, `LARAVEL_TYPE_DECLARATIONS`.
- **.php-cs-fixer.dist.php**: `@PSR12` → `@PER-CS3x0` + `@PER-CS3x0:risky`, explicit `ParallelConfigFactory::detect()`,
  removed 9 rules now subsumed by PER-CS 3.0, added `global_namespace_import`/`fully_qualified_strict_types`.
- **Code (90 sites fixed):** `! preg_match(...)` → `preg_match(...) !== 1`; `(bool) $this->option(...)` casts;
  dropped dead `glob(...) ?: []` (Filesystem::glob is `@return array`); short-ternary → long-form / hoisted var;
  native return types on 18 generator/Make* overrides (mirroring parent `@return`, **params left untyped**).
- **3 documented `ignoreErrors`** (vendor/language limits, not a baseline): `Application::path()` static-call
  false positive (path() absent from the contract, resolved via App facade `@method static`); childReturn/Param
  type in ModuleAwareGenerator (vendor GeneratorCommand loose PHPDoc); `openLock(): resource` (no native type).

## Prevention

- **Never ship `--debug`/`--sequential` as the default analyzer command.** They disable parallelism. Keep them
  in separate `*:debug` scripts. Verify a tool is actually parallel by reading its banner ("Running analysis on
  N cores").
- **Adding a native parameter type to an override of an untyped vendor parent is a PHP fatal**
  ("Declaration must be compatible"). Return-only types are legal. For Laravel `GeneratorCommand` overrides:
  add return types, keep params untyped. Already encoded in rector.php skips.
- **`StringClassNameToClassConstantRector` breaks intentional soft-dependency probes.** `class_exists('Inertia\\Inertia')`
  is a *string on purpose* — converting to `Inertia::class` forces a `use` import and trips the "loaders do not
  depend on optional UI integrations" arch test. The arch test caught it; the rule is now skipped for RouteLoader.
  When rector touches optional-integration code, re-run `composer test:arch`.
- **type-coverage error counts are inflated** — it re-emits one aggregate message per missing site. Always
  reduce to distinct (file:line) before estimating effort.
- `reportUnmatchedIgnoredErrors: true` keeps the ignore list honest — a stale ignore fails the build.
