# Test suite rewrite: Pest arch DSL + unit test coverage gaps

**Date:** 2026-05-24 12:30
**Files:** tests/Architecture/ArchitectureTest.php, tests/Unit/Manifest/ManifestValidatorTest.php, tests/Unit/Manifest/VO/FeatureSchemaTest.php, tests/Unit/Manifest/VO/ModuleTest.php, tests/Unit/Manifest/VO/FeatureValuesTest.php, tests/Unit/Manifest/VO/FeatureDefinitionTest.php, tests/Unit/Loaders/FactoryLoaderTest.php, tests/Unit/Support/ModuleLayoutTest.php, tests/Unit/Support/AtomicJsonWriterTest.php, tests/Unit/Support/ComposerNamespaceResolverTest.php, tests/Unit/Manifest/VO/ManifestMetaTest.php, tests/Unit/Manifest/VO/ManifestStateTest.php, tests/Unit/Manifest/VO/ModuleDependenciesTest.php, tests/Unit/Support/TopologicalSorterTest.php, tests/Unit/Manifest/FeatureRepositoryTest.php, phpstan.neon.dist
**Severity:** medium

## Problem

Architecture tests were verbose PHPUnit classes using Symfony Finder instead of Pest arch() DSL despite pestphp/pest-plugin-arch being installed. Several VO classes (ManifestValidator, FeatureSchema, Module) had zero unit tests. Existing tests were thin — missing round-trip verification, edge cases, and critical error paths.

## Root Cause

Tests were written during initial implementation as quick PHPUnit classes to bootstrap CI. Pest arch() DSL was added later but never migrated to. Unit test writing focused on happy paths without revisiting for coverage of error scenarios and data integrity (toArray round-trips, missing required fields, type mismatches).

## Solution

1. Replaced 8 PHPUnit architecture test files with single Pest `ArchitectureTest.php` — 13 arch tests using `arch()` DSL where supported and `test()` with file-scan/reflection for strict_types, debug calls, and static properties.
2. Created 3 new unit test files: ManifestValidatorTest (7 tests), FeatureSchemaTest (9 tests), ModuleTest (7 tests).
3. Extended 8 existing test files with 30+ additional test methods covering: toArray round-trips, error paths, edge cases (empty deps, missing fields, diamond dependencies, 3-level chains, unicode).
4. Excluded `tests/Architecture/` from PHPStan — Pest arch DSL uses dynamic method chaining PHPStan can't resolve without custom extension.

## Prevention

- When adding new VO/service classes, create test stubs immediately with at least: construction from valid data, round-trip serialization, and one error path.
- When installing a testing DSL plugin (like pest-plugin-arch), migrate existing test patterns in the same PR to avoid drift.
- PHPStan exclusion for test files using Pest DSL is expected — the tests themselves run fine; only static analysis can't resolve the dynamic API.
