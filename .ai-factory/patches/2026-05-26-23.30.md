# Extract common lifecycle test setup into CreatesLifecycleEnvironment trait

**Date:** 2026-05-26 23:30
**Files:** tests/Support/CreatesLifecycleEnvironment.php (new), 6 Unit UseCase tests, 7 Feature Command tests
**Severity:** low

## Problem

13 test files duplicated 30-60 line blocks for constructing lifecycle infrastructure:
ModuleStateRepository, ModuleManifestRepository, ModuleRegistry, ModuleRegistryCache,
ModuleDependencyGuard, LifecycleRegistryInvalidator, ModuleDirectoryPaths/Ops, ModuleSourcePreparer.
Each `makeUseCase()` or `registerServices()` method rebuilt the entire dependency graph inline.

## Root Cause

No shared extraction point for the lifecycle infrastructure factory code. Each test
independently constructed the same object graph. The `CreatesModuleFiles` trait only covered
file-level helpers (writeModuleManifest, writeModuleState), not service construction.

## Solution

Created `tests/Support/CreatesLifecycleEnvironment.php` trait with 11 composable factory methods:
- `lifecycleConfig()`, `lifecycleStateRepository()`, `lifecycleManifestRepository()`
- `lifecycleRegistryCache()`, `lifecycleRegistry()`, `lifecycleDirectoryPaths()`, `lifecycleDirectoryOps()`
- `lifecycleDependencyGuard()`, `lifecycleInvalidator()`, `lifecycleSourcePreparer()`, `lifecycleNamespaceResolver()`

Each method accepts its mandatory dependencies explicitly (no implicit cascading defaults),
keeping the composition visible in test code. Refactored all 13 test files, reducing
`makeUseCase()` methods from ~60 lines to ~15 lines and eliminating ~20 duplicate imports per file.

## Prevention

- When adding new lifecycle infrastructure tests, use `CreatesLifecycleEnvironment` instead of
  inlining object construction
- If a new infrastructure dependency is added to UseCase constructors, add a corresponding
  factory method to the trait rather than duplicating construction across tests
