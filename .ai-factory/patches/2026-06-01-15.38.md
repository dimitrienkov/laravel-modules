# Test suite hardening: shared fixtures, camelCase, attributes, arch invariants

**Date:** 2026-06-01 15:38
**Files:** tests/Support/{CreatesModuleFiles,CreatesSourceArchive,UsesTempDirectory}.php; ~75 test files across tests/Unit and tests/Feature; tests/Architecture/ArchitectureTest.php
**Severity:** low

## Problem

The test layer had drifted: PHPUnit unit/feature methods mixed `snake_case`
(`it_registers_*`, `from_manifest_*`) and `camelCase`; fixture construction
(manifest JSON shape, `state.json` writes, source-zip `ZipArchive` boilerplate,
temp-dir lifecycle) was partly in shared traits but largely duplicated locally
across ~30 files; and several documented v2.0 invariants had no enforcing test
(source artifact must not contain `state.json`; no global helpers / runtime
logging in `src/`; UseCase/Loader suffix; commandâ†’infrastructure boundary;
optional MoonShine/Inertia isolation).

## Root Cause

The suite grew iteratively. New scenarios were added faster via local private
helpers than the shared traits were extended, so there was no single owner for
fixture-building logic and no agreed method-naming convention. Documented
invariants were enforced in `src/` but the matching tests were never added in
the same change, so the contract was real but unguarded â€” drift could not be
caught mechanically.

## Solution

- Consolidated fixture builders into shared traits and reused them everywhere:
  `CreatesModuleFiles::moduleManifestArray()` / `moduleDirectory()` /
  `writeRawState()` / `readManifestFile()`; new `CreatesSourceArchive::zipModuleSource()`
  (removes 5 copies of `ZipArchive` open/add/close + manifest-encoding); and
  `UsesTempDirectory::tempPath()` / `createDirectory()` for nested-dir setup.
  Local helpers became thin scenario wrappers so scenario inputs (version,
  dependencies, schema, enabled, source) stay visible at call sites â€” helpers
  build the filesystem fixture only, they do not hide the scenario.
- Standardized all PHPUnit unit/feature methods to behavior-describing
  `camelCase` (kept `#[Test]`; Pest arch tests stay in Pest style). Domain
  `snake_case` JSON keys (`schema_version`, `display_name`, `installed_at`,
  `settings.values`) were left untouched.
- Added `#[CoversClass]` to focused unit tests and `#[Group(...)]` by area.
- Added 9 architecture assertions: UseCase/Loader suffix, namespace-wide
  command boundary, optional-integration isolation (loaders + application),
  repositoryâ†’contract implementation, and a comment-/definition-aware text scan
  forbidding global helpers (`logger`, `app`, `config`, `resolve`, ...) in `src/`.
- Closed the one genuine coverage gap (the rest of the plan's risk zones were
  already covered): a source archive carrying `state.json` is now rejected â€”
  tested at both unit (`ModuleSourcePreparer`) and feature (`modules:install`) level.

## Prevention

- Keep fixture construction in shared `tests/Support` traits; when a local
  helper duplicates a shared one, reduce it to a thin wrapper rather than copy
  the body. Reserve raw `ZipArchive`/`file_put_contents` in tests for cases
  where that artifact is the *subject* of the test (malformed/missing manifest,
  atomic-writer behavior).
- When `src/` enforces a documented invariant, add the enforcing test in the
  same change. Structural/naming/dependency invariants belong in the Pest arch
  suite (cheap, future-proof for new classes); runtime/behavioral invariants
  belong in unit/feature tests.
- For text-scan arch tests, helper names collide with method names: skip comment
  lines and exclude method definitions (`function name(`) and `->`/`::`/`$`
  prefixes, or you get false positives (e.g. `resolve()` vs
  `ApplicationNamespaceResolver::resolve()`).
- New PHPUnit method names are `camelCase` and describe the observed behavior;
  do not reintroduce `it_*`/`snake_case`.
