# Lifecycle review fixes: safety, flow, naming, tests

**Date:** 2026-05-26 22:15
**Files:** 40+ files across src/ and tests/
**Severity:** critical + medium + low (21 tasks in 5 blocks)

## Problem

Comprehensive review of `feature/lifecycle-usecases-commands` branch revealed 21 issues:
double-fault error loss, `--force` conflict with ConfirmableTrait, non-atomic permanent delete,
flow-by-exception, naming inconsistencies, dead code, TOCTOU in backup paths, direct FS calls
in UseCases, formatting gaps, and missing test coverage.

## Root Cause

Multiple independent issues accumulated during rapid lifecycle feature development:
- Error handling in catch blocks didn't account for cleanup operations also failing
- `--force` flag name collision with Laravel's ConfirmableTrait production guard
- Deletion order in permanent remove was suboptimal (directory first, state second)
- `registry->find()` + catch `ModuleNotFoundException` used where `has()` was available
- Method naming drifted from project conventions (`assert*`, `try*` patterns)
- Direct `is_dir()` calls in UseCase layer bypassed the abstraction boundary
- Backup path used only timestamp, creating TOCTOU vulnerability under parallel execution

## Solution

**Block 1 (Safety):** Wrapped rollback cleanup in try/catch for double-fault scenarios,
added makeDirectory return checks, renamed `--force` to `--yes`, introduced
`ModuleExceptionInterface` marker to narrow catch scope, reordered permanent delete
to state-first.

**Block 2 (Flow):** Replaced flow-by-exception with `has()`, unified ModuleState creation
via `updatedFrom()->withEnabled()`, extracted two private methods in RemoveModuleUseCase,
merged duplicate `findDependents` methods.

**Block 3 (Naming):** 4 renames (`tryDeleteDirectory`, `defaultDisabled`, `flushAndReset`,
`assertNoStateFile`), removed dead `meta()` method and unused `$moduleName` parameter,
added private constructors to constant holders, replaced hardcoded `0755`.

**Block 4 (TOCTOU/FS):** Added `random_bytes(4)` suffix to backup paths, added
`directoryExists()` wrapper to ModuleDirectoryOperations, simplified redundant null check.

**Block 5 (Tests):** Fixed formatting, added 2 rollback tests for UpdateModuleUseCase,
replaced chmod-based test with mock, created 3 Feature command test files (10 tests).

## Prevention

- Review catch blocks for cleanup operations that can also throw
- Avoid reusing Laravel-reserved option names (`--force` from ConfirmableTrait)
- Deletion order: remove recoverable state first, irreversible data second
- Use `has()` over try/find/catch for existence checks
- Apply random suffix to any path derived from time to prevent TOCTOU
- Keep direct PHP filesystem calls out of UseCase layer
