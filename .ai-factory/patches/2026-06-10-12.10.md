# Docs overstated loader invariants and cross-link coverage

**Date:** 2026-06-10 12:10
**Files:** docs/loaders.md, docs/module-structure.md
**Severity:** low

## Problem

Two factual inaccuracies in docs on the `docs/loaders-reference` branch, caught
before merge:

1. `docs/loaders.md` claimed "Все дефолтные лоадеры — `final readonly`", but
   `FactoryLoader` is declared `final class` (not `readonly`) in
   `src/Loaders/FactoryLoader.php`.
2. The ROADMAP entry for `docs/loaders.md` claimed cross-links from
   `architecture.md`/`module-structure.md`/README/AGENTS, but
   `docs/module-structure.md` had no link to `loaders.md` at all.

## Root Cause

1. The "all loaders are `final readonly`" sentence was written as a blanket
   generalization from the dominant pattern (14 of 15 loaders) without checking
   each class. `FactoryLoader` is the deliberate exception: it accumulates
   mutable state (model-namespace ⇒ factory-namespace map per module, the
   previously registered resolver, a one-time registration flag), so it cannot
   be `readonly`. Ironically the same doc already described this accumulation
   in "Поведенческие нюансы" — the two sections were written independently and
   never reconciled.
2. The ROADMAP progress note listed the *intended* set of cross-links rather
   than the *verified* set — the `module-structure.md` link was planned but
   never added, and the note was written from the plan, not from a grep.

## Solution

1. Rewrote the invariant sentence in `docs/loaders.md`: all default loaders are
   `final` with constructor DI; all except `FactoryLoader` are also `readonly`,
   with an anchor link to the existing behavioral-nuances explanation of why.
2. Added the missing cross-link in `docs/module-structure.md` (chosen over
   editing ROADMAP, which is read-only for `/aif-fix` and whose claim becomes
   true once the link exists): a pointer to `loaders.md` after the loader table
   and a "See Also" entry.

Verified afterwards that no other doc makes the blanket loaders-readonly claim
and that all four files named in the ROADMAP note now actually link to
`docs/loaders.md`.

## Prevention

- When a doc states a class-level invariant over a set of classes ("all X are
  Y"), verify it mechanically before writing: `grep -L "final readonly class"
  src/<dir>/*.php` instantly reveals exceptions.
- When a roadmap/progress note claims cross-links or doc coverage, derive the
  list from `grep -l` output, not from the plan.
- A genuine exception to a dominant pattern deserves an explicit "all except X,
  because…" sentence — readers extending the package copy these invariants.
