# Phase 1 review: loop order, state loss, validation gaps, priority drift

**Date:** 2026-05-24 22:00
**Files:** src/Providers/ModuleLoaderServiceProvider.php, src/Loaders/RouteLoader.php, src/Loaders/MigrationLoader.php, src/Loaders/FactoryLoader.php, src/Manifest/ManifestValidator.php, src/Manifest/FeatureDefinition.php, src/Manifest/ManifestMeta.php, src/Manifest/ManifestState.php, tests/Feature/ModuleLoaderServiceProviderTest.php, tests/Unit/Loaders/RouteLoaderTest.php, tests/Unit/Manifest/ModuleManifestRepositoryTest.php, tests/Support/ModuleFactory.php
**Severity:** critical

## Problem

Nine issues found during Phase 1 code review:
1. `bootModules()` iterates modules-first, loaders-second — violates ARCHITECTURE.md priority guarantees when >1 module exists.
2. `FactoryLoader` registered as transient (`bind`), losing accumulated namespace mappings across iterations.
3. Multi-module test used only 1 enabled module, masking the loop order bug.
4. `RouteLoader` missing `routesAreCached()` guard — routes registered twice when cache is active.
5. `ManifestValidator::requiredObject()` rejects valid empty `{}` because `array_is_list([])` returns `true`.
6. `FeatureDefinition::ALLOWED_KEYS` missing `label`/`description`/`group` needed for MoonShine settings UI.
7. `ManifestMeta`/`ManifestState` missing `license`/`updated_at` fields documented in ARCHITECTURE.md.
8. Loader priorities diverged from ARCHITECTURE.md table (Migration=40, Factory=50, Route=60 vs spec 30/31/50).

## Root Cause

Each issue had a distinct root cause:
- Loop order: implementation didn't match the architecture spec (loaders-first, modules-second).
- State loss: `bind()` creates new instances on every `tagged()` resolution — `FactoryLoader` accumulates state across `load()` calls but got fresh instances each time.
- Single-module test: both loop orderings produce identical output with 1 module — bug was invisible.
- Missing cache guard: `routesAreCached()` is on `Illuminate\Foundation\Application`, not the contract — easy to miss when injecting the interface.
- Empty object validation: PHP's `array_is_list([])` returning `true` is a well-known gotcha — `json_decode('{}')` produces `[]`.
- Missing manifest fields: ARCHITECTURE.md §4 documents them, but they weren't in the ALLOWED_KEYS whitelists.
- Priority drift: values were assigned incrementally during development without cross-checking the architecture table.

## Solution

1. Flipped loop nesting in `bootModules()` to loaders-first, modules-second. Cached loaders array before the loop.
2. Changed all loader registrations from `bind()` to `singleton()` — safe because loaders are stateless by contract (ARCHITECTURE §13); `FactoryLoader` is the sole stateful one, and singleton guarantees correct accumulation.
3. Added second enabled module + disabled module to pipeline test. Expected order now proves loaders-first ordering.
4. Added `Illuminate\Foundation\Application` (concrete class, not contract) to `RouteLoader` constructor + early return when `routesAreCached()` is true.
5. Changed `requiredObject()` condition from `!is_array || array_is_list` to `!is_array || ($value !== [] && array_is_list($value))`.
6. Added `label`/`description`/`group` as optional string fields to `FeatureDefinition` with parsing and serialization.
7. Added `license` to `ManifestMeta` and `updated_at`/`updatedAt` to `ManifestState` with parsing and serialization.
8. Aligned priorities: Migration=30, Factory=31, Route=50.

## Prevention

- **Multi-module tests mandatory**: any test covering the loader pipeline must use 2+ enabled modules to detect loop ordering bugs.
- **Singleton vs bind audit**: when a loader accumulates state across `load()` calls, `bind()` silently breaks it. Default to `singleton` for pipeline loaders.
- **`array_is_list([])` gotcha**: when validating "must be associative array", always handle the empty array edge case explicitly.
- **ARCHITECTURE.md as source of truth for priorities**: after changing a `priority()` return value, grep the architecture doc table to verify alignment.
- **Whitelist audit on new manifest fields**: when ARCHITECTURE.md adds a field, immediately check all `ALLOWED_KEYS` arrays.
- **Route cache guard**: any loader that registers routes must check `routesAreCached()` — add to code review checklist.
