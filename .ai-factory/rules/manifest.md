# Правила манифеста

> Правила для работы с `module.json`. Загружаются после `rules/base.md`.

## Правила

- `module.json` остаётся единственным источником правды для метаданных, состояния, схемы фичетоглов и значений фичетоглов; не добавляй хранение состояния модулей в БД.
- `meta` и `settings.schema` неизменяемы после установки; меняй их только через `modules:update`.
- В обычных runtime-операциях изменяй только `state` и `settings.values`, всегда через `ModuleManifestRepository`.
- Валидируй `settings.values` по `settings.schema` при каждой записи манифеста.
- Не добавляй секцию `autoload` в `module.json`; лоадеры находят работу по файлам и директориям модуля через `ModuleLayout`.
- Нормализуй короткий список зависимостей в `moduleName => "*"` и храни зависимости как Composer SemVer constraints.
