### Laravel Loader Lifecycle Checklist
**Source**: 2026-05-24-21.30, 2026-05-24-22.00, 2026-05-25-14.56, 2026-05-25-22.45
**Rule**: When reviewing loader code, verify all of the following:
- Deferred service integration uses `ContainerLifecycleHooks::callAfterResolving()`, not plain `afterResolving()` or `$app->bound()`.
- Loaders are registered as `singleton`, not `bind` — `bind()` silently breaks state accumulation across `load()` calls.
- Route-registering loaders check `routesAreCached()` via `Illuminate\Foundation\Application`.
- Loaders do not constructor-inject heavy deferred framework services (`migrator`, `translator`, `view`, `BladeCompiler`, `BroadcastManager`).
- Global static resolver setters (e.g., `Factory::$factoryNameResolver`) preserve the previous resolver and delegate unrelated input.
- Optional service loading uses `callAfterResolving` rather than `$app->booted()`.
- Reflection on framework internals is wrapped in `try/catch(ReflectionException)`.

### Validation and Manifest Checklist
**Source**: 2026-05-24-22.00, 2026-05-25-00.30, 2026-05-25-13.45
**Rule**: When reviewing validation and manifest code, check:
- `array_is_list([])` returns `true` — any "must be associative" validation must handle empty arrays explicitly.
- Character-length validation uses `mb_strlen($v, 'UTF-8')`, not `strlen()`.
- Identifier validation uses the shared regex from `ManifestFieldReader`, not ad-hoc patterns.
- No silent normalization (`array_unique()`, type coercion) — prefer explicit validation errors.
- New manifest fields in ARCHITECTURE.md are reflected in all `ALLOWED_KEYS` arrays.
- Collection validation checks whether item state exempts it from some checks (e.g., disabled modules).

### Filesystem and Atomicity Checklist
**Source**: 2026-05-25-13.45, 2026-05-25-14.56, 2026-05-25-17.30, 2026-05-25-22.45
**Rule**: When reviewing filesystem code, check:
- Return values of `file_put_contents()`, `unlink()`, `rename()` are checked and failures produce typed exceptions.
- Atomic writes use `AtomicFileWriter` from `src/Support/` — flag duplicated atomic write logic.
- Temp and lock files are cleaned up in the same `finally` block that creates them.
- No TOCTOU patterns: flag `is_file()` followed by `unlink()` — prefer `@unlink()` + re-check.
- New file I/O implementations check `src/Support/` for existing shared utilities before reimplementing.
- Infrastructure sharing uses composition with default constructor parameters, not inheritance.

### Architecture and Structure Checklist
**Source**: 2026-05-24-21.30, 2026-05-25-00.30, 2026-05-25-01.25, 2026-05-25-13.45
**Rule**: When reviewing architecture and structure, flag:
- Classes with 3+ distinct responsibilities — should be split into role-based classes.
- Production cache (`bootstrap/cache/modules.php`) containing mutable runtime data (e.g., `settings.values`) — cache must contain only immutable discovery data.

- Module config/routes merged into shared namespace without scoping under module identity.
