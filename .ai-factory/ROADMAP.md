# Roadmap пакета `dimitrienkov0/laravel-modules`

> Модульная Laravel-платформа для коммерческой поставки модулей заказчикам: манифест `module.json`, loader-pipeline, фичетоглы, опциональный MoonShine admin-UI, Octane-safe. Версия `2.0` — мажорный рефакторинг ядра без обратной совместимости с `1.x`.

## Milestones

### Фаза 0 — Актуализация репозитория и тулинга (фундамент)
- [ ] **Обновление composer-зависимостей** — поднять до актуальных линейок: `laravel/framework ^11|^12|^13` (последние патчи), `moonshine/contracts ^4`, `moonshine/core ^4`, `orchestra/testbench ^9|^10|^11`, `phpunit/phpunit ^11|^12`, `phpstan/phpstan ^2`, `larastan/larastan ^3` (заменить `nunomaduro/larastan ^2`), `rector/rector ^2`, `driftingly/rector-laravel ^2`, `friendsofphp/php-cs-fixer ^3` (последний), добавить `pestphp/pest ^3` и `pestphp/pest-plugin-arch ^3`. Запустить `composer update`, зафиксировать `composer.lock`.
- [ ] **Конфиги инструментов качества** — `phpstan.neon` (level 8, larastan, `treatPhpDocTypesAsCertain: false`, кастомное правило-запрет `Illuminate\Database\Eloquent\Model` в `Application/UseCases/*`), `rector.php` (Laravel-set + type-declaration + dead-code + early-return), `.php-cs-fixer.dist.php` (`@PSR12` + strict-types declare + ordered-imports + проектные правила), `phpunit.xml` с testsuites `Architecture`/`Unit`/`Feature`, `Pest.php` bootstrap.
- [ ] **Composer scripts и CI workflow** — `composer scripts`: `test`, `test:arch`, `test:unit`, `test:feature`, `phpstan`, `rector:dry`, `format`, `format:dry`. GitHub Actions workflow на матрицу `php: [8.3, 8.4] × laravel: [11, 12, 13]`: phpstan + rector:dry + format:dry + полный test-набор обязательны.

### Фаза 1 — v2.0 ядро (переписать уже реализованное)

- [ ] **Контракты и манифест** — `LoaderInterface` (только `load(Module)` + `priority()`), `ModuleRegistryInterface`, `ModuleManifestRepositoryInterface`, `FeatureRepositoryInterface`, `NamespaceResolverInterface`, `ManifestValidatorInterface`. JSON-схема `module.json` **без секции `autoload`** (pure convention over configuration) — только `meta`, `state`, `settings.schema`, `settings.values`. Value-объекты: `Module`, `ManifestMeta`, `ManifestState`, `ModuleDependencies`, `FeatureSchema`, `FeatureDefinition`, `FeatureValues` — все `final readonly`.
- [ ] **Support-утилиты и исключения** — `ModuleLayout` (`final readonly`, единая точка истины путей модуля: `routesDir`, `migrationsDir`, `factoriesDir`, `configDir`, `langDir`, `viewsDir`, `bladeComponentsDir`, `commandsDir`, `consoleRoutesFile`, `channelsFile`, `providersDir`, `observersDir`, `policiesDir`, `middlewareDir`), `AtomicJsonWriter` (tmp + rename + flock), `ComposerNamespaceResolver` (PSR-4 из корневого `composer.json`, без хардкода `App\\`), `TopologicalSorter` для `meta.dependencies`, `ZipExtractor`, типизированные исключения (`InvalidManifestException`, `ModuleNotFoundException`, `ModuleDependencyMissingException`, `CyclicDependencyException`, `ManifestWriteException`).
- [ ] **ModuleRegistry и ModuleManifestRepository** — singleton-сканер `paths.directories` с прогревом на boot worker'а, чтение/валидация/атомарная запись `module.json`, топологическая сортировка, чтение прогретого кеша из `bootstrap/cache/modules.php` одним `require` без FS-операций. Никакого mutable-state кроме private `?array $cache = null`.
- [ ] **FeatureRepository (Octane-safe runtime)** — `scoped`-биндинг, per-request in-memory кеш, читает `module.json` напрямую через `ModuleManifestRepository` (не из `modules.php`-кеша) — чтобы MoonShine-правки `settings.values` подхватывались следующим request'ом без `optimize-clear`. API: `get/bool/int/string`.
- [ ] **Новый ServiceProvider с pipeline** — переписать `ModuleLoaderServiceProvider`: `register()` биндит контейнер (singleton/scoped по таблице из ARCHITECTURE §12), `boot()` делает один проход `foreach loader in sortedByPriority { foreach module in loadOrder { if enabled → loader->load($module) } }`. MoonShine — только через `afterResolving(CoreContract::class)`. Удалить двойной `ConfigLoaderService::autoload()` из `register/boot`.
- [ ] **Миграция уже реализованных лоадеров на новый контракт** — `ConfigLoader` (Config/*.php merge), `RouteLoader` (config-driven `modules.routing.types`, версионирование через `Routes/api/v1.php`, `Routes/api/v2.php` → prefix `api/v1`, `api/v2`), `MigrationLoader`, `FactoryLoader`, `ServiceProviderLoader` (convention `Providers/*ServiceProvider.php`, без classmap-обхода). Удалить старые `Services/*Service.php` и старый `ModuleLoaderServiceProvider`.
- [ ] **MoonShineLoader без classmap-магии** — тонкая обёртка `$core->autoload($module->namespace)` для каждого включённого модуля; регистрация на `afterResolving(CoreContract::class)`. Удалить `discoverPages/discoverResources` и reflection-обходы из 1.x.
- [ ] **Архитектурные тесты на ядро** — Pest+arch: `final readonly` у VO в `Manifest/`, `final` у Loader'ов и сервисов, все классы из `Loaders/` реализуют `LoaderInterface`, запрет `Illuminate\Support\Facades\*` вне `Console/Commands` и `MoonShine`, запрет `static` properties в `src/`, запрет `Illuminate\Database\Eloquent\Model` в `Application/UseCases/*`. Прогон в CI обязателен.

### Фаза 2 — v2.0 расширение (новые функции)

- [ ] **Новые лоадеры** — `LangLoader` (`Lang/{locale}/*.php` через `loadTranslationsFrom`, namespace = `Str::snake($module->name)`), `ViewLoader` (`Resources/views/` через `loadViewsFrom`), `BladeComponentLoader` (`View/Components/` через `Blade::componentNamespace`), `CommandLoader` (class-based из `Console/Commands/`), `ConsoleRouteLoader` (`Routes/console.php` в стиле Laravel 11+: `Artisan::command(...)` + `Schedule::command(...)`), `EventLoader` (`Providers/EventServiceProvider.php` или auto-discovery), `ObserverLoader` (`Domain/Observers/` по соглашению `Model → ModelObserver`), `PolicyLoader` (`Domain/Policies/` аналогично), `MiddlewareLoader` (`Http/Middleware/` с named-регистрацией), `BroadcastLoader` (`Routes/channels.php`).
- [ ] **Application/UseCases жизненного цикла** — `InstallModuleUseCase`, `UpdateModuleUseCase`, `RemoveModuleUseCase`, `EnableModuleUseCase`, `DisableModuleUseCase`, `ScaffoldModuleUseCase`. Все — `final readonly` с DI, без фасадов, без хелперов.
- [ ] **Artisan-команды жизненного цикла** — тонкие обёртки над UseCase: `modules:make`, `modules:list`, `modules:enable`, `modules:disable`, `modules:install <zip|path>` (валидация манифеста до распаковки, dependency check, `InstallHook::handle`), `modules:update` (бэкап `settings.values` + merge), `modules:remove` (откат миграций, перенос в `backup/`), `modules:optimize` / `modules:optimize-clear` (формат `bootstrap/cache/modules.php` с готовым `Module[]` + `load_order`, регистрация через `$this->optimizes(...)`).
- [ ] **Генераторы `make:module-*`** — полный набор по ARCHITECTURE §2: controller (`-mfsr`), model, usecase, action, query, dto, enum, event, listener, observer, policy, middleware, request, resource (Http), migration, factory, seeder, command, moonshine-resource, moonshine-page. Stubs в `src/stubs/`, единый базовый класс генератора с резолвом namespace через `ComposerNamespaceResolver`.
- [ ] **MoonShine admin-UI** — `ModulesResource` (список модулей с переключателем enabled/disabled и действиями install/update/remove на in-memory коллекции из `ModuleRegistry::all()`, без модели Eloquent), `ModuleSettingsPage` (динамическая форма по `settings.schema`: `bool→Switcher`, `int→Number` с min/max, `enum→Select`, `string→Text`), запись через `ModuleManifestRepository::updateFeatureValues(FeatureValues $values)` (никаких сырых array на границе).
- [ ] **Кеш registry для production** — формат `bootstrap/cache/modules.php` с массивом сериализованных `Module` + `load_order`, интеграция с Laravel `optimize`/`optimize:clear` через `$this->optimizes()`. `ModuleRegistry` читает кеш одним `require`. `FeatureRepository` намеренно мимо кеша — читает `module.json` напрямую.
- [ ] **Unit и Feature тесты** — PHPUnit + Orchestra Testbench: unit-тесты на каждый Loader (через `Module`-фабрику), `ModuleManifestRepository` (atomic write, schema validation, file locking), `TopologicalSorter` (циклы, плоский граф, изолированные модули), `ComposerNamespaceResolver`, `AtomicJsonWriter`, `ModuleLayout`. Feature-тесты команд `modules:install/update/remove/enable/disable/list/optimize`. Feature-тесты `FeatureRepository` (повтор чтения в рамках request'а, сброс между request'ами через container scoped). MoonShine-тесты пропускаются при отсутствии MoonShine.
- [ ] **Octane-совместимость и архитектурные гарантии** — прогон под Swoole/RoadRunner/FrankenPHP с проверкой отсутствия leak'ов: state сервисов не накапливается, `FeatureRepository::$cache` сбрасывается между запросами, `ModuleRegistry` остаётся immutable после первого скана. Архитектурный тест `FeatureRepository` ≠ singleton. Документация требований к биндингам в `Octane.md`.
- [ ] **Документация и миграция** — `docs/architecture.md`, `docs/manifest.md`, `docs/loaders.md` (включая создание собственных), `docs/commands.md`, `docs/moonshine.md`, `docs/octane.md`, `docs/migration-from-1x.md` (ручная миграция, потому что BC нет). Обновлённый README как landing-page со ссылками.

### Фаза 3 — v2.1+ (расширения после стабилизации ядра)

- [ ] **События жизненного цикла модуля** — `ModuleInstalled`, `ModuleUpdated`, `ModuleRemoved`, `ModuleEnabled`, `ModuleDisabled`, `FeatureValueChanged`; хост-приложение подписывается через стандартный Laravel EventServiceProvider без модификации пакета.
- [ ] **Подпись манифестов** — опциональная криптографическая подпись `module.json` и архива (Ed25519 или GPG) для защиты канала поставки модулей заказчику; верификация в `modules:install` до распаковки.
- [ ] **Marketplace и удалённая поставка** — формат пакета (zip + manifest + signature), CLI `modules:fetch <url>` и `modules:upgrade-all` с проверкой совместимости версий, опциональный приватный реестр через HTTP/S3.
- [ ] **i18n для `settings.schema`** — `label`/`description`/`group` через ключи переводов из `Lang/{locale}/`, MoonShine-форма локализует UI по текущей локали приложения.
- [ ] **Расширенные типы фичетоглов** — `multiselect`, `json`, `file`, `relation` (с авто-резолвом через DI), валидация через Laravel-валидатор поверх JSON Schema.
- [ ] **CLI для разработчика модулей** — `modules:doctor` (диагностика манифеста, зависимостей, структуры папок), `modules:diff <name> <version>` (что изменится при update), `modules:pack <name>` (zip-сборка модуля с опциональной подписью).
